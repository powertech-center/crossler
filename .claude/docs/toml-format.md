## Формат TOML — краткий, но полный гид

TOML (Tom's Obvious, Minimal Language) — простой конфигурационный формат, ориентированный на читаемость человеком и однозначное отображение в примитивные типы.

> **Версия:** этот документ описывает **TOML 1.1**. Crossler использует библиотеку `github.com/BurntSushi/toml`, которая поддерживает TOML 1.1. Возможности, доступные только в 1.1 (и недоступные в более распространённом 1.0), явно помечены в тексте.

```toml
title = "Пример TOML"
version = 1.0
debug = true
```

---

## Базовый синтаксис

- **Файл**: обычный текст в UTF‑8.
- **Ключ = значение**: одна пара на строку.
- **Комментарии**: начинаются с `#` и длятся до конца строки.
- **Пробелы** вокруг `=` не обязательны, но желательны для читаемости.

```toml
# Комментарий
name = "Service"
port=8080   # так тоже допустимо
```

Ключи по умолчанию принадлежат «корневой» таблице (глобальной секции файла).

---

## Ключи

### Простой ключ

Состоит из букв, цифр, `_` и `-`, не начинается с цифры.

```toml
host = "localhost"
max_retries = 5
```

### Кавычки в ключах

Если нужны пробелы, точки или спецсимволы, ключ берётся в кавычки. Допустимы как двойные (`"`), так и одинарные (`'`) кавычки.

```toml
"listen port" = 8080
"db.url" = "postgres://user:pass@localhost/db"
'it''s fine' = true   # одинарные — raw, без escape
```

Допустим пустой ключ (редко, но валиден):

```toml
"" = "пустой ключ"
```

С кавычками допустим любой Unicode:

```toml
"версия" = "1.0"
```

### Точечные (dotted) ключи

Точка разделяет уровни вложенных таблиц. Эквивалентно объявлению вложенных таблиц.

```toml
server.host = "127.0.0.1"
server.port = 8080

# То же самое:
[server]
host = "127.0.0.1"
port = 8080
```

### Смешение dotted-ключей и `[table]` — ловушка

После того как таблица объявлена через `[header]`, нельзя добавлять ключи к ней через dotted-синтаксис **выше** по файлу. Но можно расширять **ниже**, если таблица объявлена через dotted-ключи:

```toml
# Допустимо: dotted-ключ, затем [table] как вложение
app.name = "demo"

[app.logging]   # расширяет app, не переопределяет
level = "info"
```

```toml
# Ошибка: смешение стилей в одной «области видимости»
[server]
host = "localhost"
server.port = 8080  # нельзя: server уже открыт через [server]
```

Правило простое: **внутри секции `[table]` используйте только простые ключи**, dotted-ключи — только на уровне корня или до открытия соответствующего `[header]`.

## Типы данных

### Строки

1. **Обычные** (escape‑последовательности работают):

```toml
path = "C:\\Program Files\\App"
text = "Строка\nс переводом строки"
```

Полный список escape-последовательностей:

| Последовательность | Значение |
|--------------------|----------|
| `\\` | обратный слеш |
| `\"` | двойная кавычка |
| `\n` | перевод строки (LF) |
| `\r` | возврат каретки (CR) |
| `\t` | табуляция |
| `\b` | backspace |
| `\f` | form feed |
| `\uXXXX` | Unicode (4 hex-цифры) |
| `\UXXXXXXXX` | Unicode (8 hex-цифр) |
| `\e` | ESC (0x1B) — **только TOML 1.1** |
| `\xHH` | ASCII hex (2 цифры) — **только TOML 1.1** |

2. **Буквальные** (raw, escape не работают):

```toml
windows_path = 'C:\Program Files\App'
regex = '\d{3}-\d{2}-\d{2}'
```

3. **Многострочные**:

```toml
text = """
Первая строка
Вторая строка
"""

raw = '''
Без экранирования
и с \n как текстом
'''
```

### Целые числа

Десятичные, можно использовать `_` как разделитель.

```toml
count = 42
big = 1_000_000
negative = -5
```

Шестнадцатичные, восьмеричные, двоичные:

```toml
hex = 0xFF
oct = 0o755
bin = 0b1010
```

**Ведущие нули запрещены** — чтобы не путать с восьмеричными:

```toml
bad = 007   # Ошибка парсинга
good = 7    # Правильно
```

### Числа с плавающей точкой

```toml
ratio = 0.1
pi = 3.14159
exp1 = 1e6
exp2 = -2.5e-3
nan_value = nan
inf_value = inf
neg_inf = -inf
```

### Логические значения

```toml
enabled = true
debug = false
```

### Дата и время

1. **Полный `offset datetime`** (предпочтительный тип для «точного момента во времени»):

```toml
release_at = 2024-01-02T15:04:05+03:00
utc_time = 2024-01-02T12:04:05Z
```

2. **`local datetime`** (без часового пояса):

```toml
local_time = 2024-01-02T15:04:05
```

3. **`local date`**:

```toml
holiday = 2024-05-09
```

4. **`local time`**:

```toml
start_at = 09:30:00
```

## Массивы

Список значений одного типа (строго рекомендуется не смешивать типы).

```toml
ports = [8080, 8081, 8082]
hosts = ["a.example.com", "b.example.com"]
flags = [true, false, true]
```

Допускается перенос и запятая после последнего элемента:

```toml
numbers = [
  1,
  2,
  3,
]
```

---

## Таблицы (tables)

Таблица — это словарь ключ‑значение. Объявляется через заголовок в квадратных скобках.

```toml
[server]
host = "localhost"
port = 8080
```

### Вложенные таблицы

```toml
[database]
user = "app"

[database.auth]
method = "password"
timeout = 30
```

Нельзя переопределять уже полностью объявленные таблицы, кроме расширения новыми ключами.

```toml
[app]
name = "demo"

[app]        # Ошибка: повторное объявление той же таблицы
version = "1.0"
```

Правильно:

```toml
[app]
name = "demo"
version = "1.0"
```

### Порядок

- Ключи внутри таблицы могут идти в любом порядке.
- Таблицы должны располагаться после всех простых ключей, которые они используют как префикс.

```toml
app.name = "demo"

[app.logging]  # допустимо: таблица после dotted‑ключа
level = "info"
```

## Встроенные таблицы (inline tables)

Короткая запись таблицы в одну строку.

```toml
database = { host = "localhost", port = 5432, user = "app" }
```

**В TOML 1.0** inline table обязана быть в одну строку и **неизменяема** после объявления — нельзя добавить ключи снаружи:

```toml
server = { host = "localhost" }
server.port = 8080  # Ошибка в TOML 1.0: нельзя расширять inline table
```

**В TOML 1.1** разрешён многострочный синтаксис и расширение:

```toml
# TOML 1.1 — многострочная inline table:
service = {
  name = "api",
  limits = { rps = 100, burst = 200 },
}

# TOML 1.1 — расширение inline table снаружи:
server = { host = "localhost" }
server.port = 8080  # допустимо
```

Подходят для небольших структур; для крупных удобнее обычные таблицы.

---

## Массивы таблиц (array of tables)

Используются для списков однотипных объектов.

```toml
[[service]]
name = "auth"
port = 8000

[[service]]
name = "api"
port = 8001
```

Вложенные таблицы внутри массива таблиц:

```toml
[[server]]
name = "eu-central"

  [server.tls]
  enabled = true
  min_version = "1.2"

[[server]]
name = "us-east"

  [server.tls]
  enabled = false
```

`[[server]]` создаёт новый элемент в массиве, `[server.tls]` относится к «текущему» элементу.

---

## Комментарии и пустые строки

- Комментарии начинаются с `#`.
- Можно размещать после значения.
- Пустые строки игнорируются и служат только для читаемости.

```toml
# Общие настройки
log_level = "info"  # может быть debug/info/warn/error


# Настройки БД
[database]
host = "localhost"
```

## Типичные паттерны использования

### Конфиг веб‑сервиса

```toml
[server]
host = "0.0.0.0"
port = 8080

[logging]
level = "info"
format = "json"

[database]
url = "postgres://user:pass@localhost:5432/app"
max_open = 10
max_idle = 5
```

### Несколько окружений

```toml
[env.dev]
debug = true
db_url = "postgres://dev:dev@localhost/dev"

[env.prod]
debug = false
db_url = "postgres://prod:prod@db/prod"
```

### Список воркеров

```toml
[[worker]]
name = "worker-1"
queue = "emails"

[[worker]]
name = "worker-2"
queue = "reports"
```

## Ограничения и лучшие практики

- **Нет вычислений**: TOML — только данные, без переменных и выражений.
- **Не смешивайте типы в массивах**, чтобы избежать сюрпризов в разных библиотеках.
- **Явно задавайте часовой пояс** для моментов времени (используйте `offset datetime`).
- **Не дублируйте объявления таблиц** и не перемешивайте структуры: сначала спланируйте иерархию ключей.
- **Не используйте ведущие нули** в целых числах — `007` является синтаксической ошибкой.
- **Внутри `[table]`-секции не используйте dotted-ключи** для той же таблицы — это источник неочевидных ошибок.
- **Учитывайте версию TOML**: многострочные inline tables и расширение inline tables — только TOML 1.1; большинство конфигов совместимы с 1.0.
- Для больших конфигов группируйте параметры по подсистемам (`[server]`, `[database]`, `[auth]` и т.д.).

---

## Резюме

- TOML даёт простое и строго типизированное представление конфигурации.
- Опорные конструкции: **ключи**, **таблицы**, **массивы**, **даты и время**, **массивы таблиц**.
- При следовании базовым правилам (не смешивать типы, не дублировать таблицы, явно задавать время) TOML остаётся легко читаемым и надёжно парсится практически на любом языке.
