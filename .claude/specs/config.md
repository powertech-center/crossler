# Crossler — Config Format

Рабочий документ по проектированию формата конфигурационного файла и CLI.
Содержимое — текущее понимание и ориентиры, не финальная спецификация.

---

## Группы данных конфига

Предварительный список логических групп. Порядок и состав будут уточняться.

### 1. `metadata`
Общая информация о продукте: название, версия, описание, лицензия, мейнтейнер, сайт и т.д.

### 2. `binaries`
Пути к бинарникам по таргетам (ОС + архитектура), куда их устанавливать.
Выделены отдельно, потому что:
- подписываются *до* упаковки
- имеют платформо-зависимое расширение (`.exe` на Windows)
- путь к артефакту сборки зависит от таргета

### 3. `files`
Все остальные файлы, симлинки, директории.
Каждый элемент описывает: источник, путь назначения, тип (`file`, `symlink`, `dir`), атрибуты (`config`, `readonly`, `noreplace` и т.д.).
Симлинки — тип файла, не отдельная группа.
Флаги — атрибут конкретного файла, не отдельная группа.

### 4. `scripts`
Хуки жизненного цикла пакета: preinstall, postinstall, preremove, postremove.

### 5. `dependencies`
Зависимости от системных пакетов. В первую очередь актуально для Linux (`.deb`, `.rpm`).

### 6. `signing`
Параметры подписания артефактов:
- Windows Authenticode (через `osslsigncode` или `signtool`)
- Apple Code Signing + нотаризация (через `codesign`/`notarytool` или `rcodesign`)

### 7. `integration`
Системная интеграция. Внутри — подсекции:
- сервисы / демоны (systemd, launchd, Windows Service)
- переменные окружения
- ярлыки (Windows shortcuts)
- контекстное меню

### 8. `platforms`
Переопределения для конкретных ОС (`linux`, `macos`, `windows`) или форматов пакетов (`deb`, `rpm`, `msi` и т.д.).
Базовые настройки работают для всех платформ; здесь — только то, что нужно изменить.
